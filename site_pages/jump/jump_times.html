Focus Area Question Answers
Identify parts What variables can you find in this program? (What are the names of the values that store data?)
Typical variables include timesTable (the number whose table you’re generating), x (the counter in the loop) and answer (the result of multiplying the two numbers).
Understand process What does the for x in range(1,11): line do? (How many times will the loop run?)
for x in range(1,11): repeats the code inside the loop 10 times, with x taking the values 1, 2, 3, …, 10 in turn.
Explore change What would happen if you changed the range to (1,6) instead of (1,11)? (How would that affect the output?)
The loop would run 5 times (x = 1 to 5), so you would only see the first five rows of the times table instead of 10.
Output formatting What does the f in print(f"...") do? (How does it help display variable values in the text?)
The f makes it an f-string, which lets you put variables straight into the text using {curly_braces}. For example, print(f"{x} x {timesTable} = {answer}") shows the actual values of x, timesTable and answer.
Debug What kind of error might happen if you removed answer = 0 at the start? (Would the program still run correctly? Why or why not?)
If the program tries to use answer before it is given a value, Python will raise a NameError (variable not defined). Initialising it with answer = 0 makes sure it exists before the loop starts.
Predict & Reason What would happen if timesTable was a word instead of a number? (Think about what Python would do when it tries to multiply.)
If timesTable is a string (a word) and the code expects a number, you’ll either get a type error or strange behaviour. For example, "3" * 4 repeats the text ("3333") instead of doing a multiplication, and mixing a word with a number (like "cat" * "3") will cause an error.
