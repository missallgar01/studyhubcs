Inputs, Processes & Outputs

What is an input?

  • An input is data or information that is entered into a program before it is processed.
  • Inputs can come from different sources, such as:
    • User input – keyboard, mouse, controller, microphone
    • Sensors – temperature, pressure, movement
  • Without inputs, programs cannot interact with the outside world and will always produce the same result.

What is a process?

  • A process is an action carried out in an algorithm that transforms inputs into outputs.
  • The CPU executes the instructions that make up the process.
  • Examples of processes include:
    • Comparing two numbers
    • Calculating an average

What is an output?

  • An output is the result of processing in an algorithm.
  • Outputs allow the user to see whether a program is working correctly.
  • Outputs can include:
    • Numbers – results of calculations
    • Text
    • Images
    • Actions – triggering events

Repetition – FOR Loops (Python)

Repetition allows a program to repeat a block of code multiple times. A FOR loop is a count-controlled loop, meaning it runs a fixed number of times.

Repetition Python
FOR loop
(count controlled)		 

for x in range(10):
    print("Hello")
This will print the word "Hello" 10 times (0–9 inclusive)
FOR loop with step 

for x in range(2, 12, 2):
    print(x)

# Python range function excludes end value
This will print the even numbers from 2 to 10 inclusive
FOR loop counting backwards 

for x in range(10, -1, -1):
    print(x)

# Python range function excludes end value
This will print the numbers from 10 to 0 inclusive

The range() function controls how many times the loop runs and can take:

  • One value: end value
  • Two values: start and end
  • Three values: start, end, and step

Selection

What is selection?

  • Selection is when the flow of a program is changed depending on a set of conditions.
  • The result of a condition determines which lines or blocks of code are run next.
  • Selection is commonly used for validation, calculations, and responding to a user’s choices.

Ways to write selection statements:

  • IF… THEN… ELSE – conditions are tested sequentially
  • CASE / SWITCH – an expression is tested against multiple constant values

Programming Functions & Procedures

What are functions and procedures?

  • Functions and procedures are a type of subprogram – a sequence of instructions that performs a specific task.
  • Subprograms are used to simplify programs by breaking them into smaller, manageable parts.
  • Subprograms can be used to:
  • Avoid duplicating code and allow code to be reused
  • Improve the readability and maintainability of programs
  • Perform calculations, retrieve data, or make decisions
  • Parameters are values passed into a subprogram.
  • Parameters appear inside brackets after the subprogram name.
  • Example: taxCalculator(pay, taxCode)
  • Subprograms can have multiple parameters.
  • To use a subprogram, it must be called from the main program.

What is the difference between a function and a procedure?

  • A function returns a value.
  • A procedure does not return a value.

Worked Example

Question

An economy-class airline ticket costs £199. A first-class airline ticket costs £595.

(A) Create a function, flightCost(), that takes the number of passengers and the type of ticket as parameters, calculates and returns the total price to pay.

(B) Write program code that uses flightCost() to output the price of 3 passengers flying economy.

You do not need to validate the parameters.

Answer

Part A – Creating the function


def flightCost(passengers, ticketType):
    if ticketType == "economy":
        cost = 199 * passengers
    elif ticketType == "first":
        cost = 595 * passengers
    return cost

Part B – Calling the function


print(flightCost(3, "economy"))

# OR

total = flightCost(3, "economy")
print(total)

Records & Databases

What is a database?

  • A database is an organised collection of data

Examiner Tips and Tricks – Text Files

When opening files in a program, it is very important to use the correct file mode in the open() command.

Mode Meaning What it does
"r" Read Opens a file for reading only. The file must already exist.
"w" Write Opens a file for writing. If the file does not exist, it is created. If it does exist, the contents are overwritten.

Types of Errors

Error type Definition What happens? Examples
Syntax Error An error that breaks the rules or grammar of a programming language. The program will not run.
  • Typos or spelling mistakes
  • Missing or extra brackets or quotes
  • Incorrect indentation
  • Invalid variable or function names
  • Incorrect use of operators
Logic Error Incorrect code that allows the program to run but produces an incorrect result. The program runs but gives the wrong output.
  • Using < instead of >
  • Using AND instead of OR
  • Looping one extra time
  • Indexing arrays incorrectly (starting from 1 instead of 0)

  • Index out of range in an array
  • Unable to read from or write to a drive

Identifying Logic Errors

Logic errors can be harder to identify than syntax errors because the program still runs but does not produce the expected output.

Why logic errors are difficult to find:

  • The program runs without crashing
  • The output may look correct for some inputs
  • Errors often only appear with boundary or erroneous data

Common areas to check for logic errors:

  • Relational operators (<, >, ==, !=)
  • Logical operators (AND, OR, NOT)

Logic Error Example

Incorrect code (logic error):


age = int(input("Enter your age: "))

if age > 11 or age < 18:
    print("You are now in secondary school!")
else:
    print("You are not in secondary school!")

The condition uses OR, which means the condition is almost always true. As a result, the else statement is never reached.

For example, entering an age of 21 incorrectly produces: 


You are now in secondary school!

Corrected Code


age = int(input("Enter your age: "))

if age > 11 and age < 18:
    print("You are now in secondary school!")
else:
    print("You are not in secondary school!")

Changing OR to AND fixes the logic error. The program now correctly checks that the age is within the required range.

This logic error would be identified by testing with boundary values such as 11, 18, or values outside the range.

Programming Validation

Validation is code used to check that data entered by a user is acceptable and meets the requirements of a program.

More than one type of validation can be applied to the same input. Validation is often implemented using while loops so the program repeats until valid data is entered.

Validation Type What it checks Example use Python Example
Length check Checks the length of a string. Ensuring a password is at least 8 characters long. 

while len(password) < 8:
    password = input("Enter a password (8+ characters)")
Range check Ensures a number falls within a specific range. Checking an age is between 0 and 100. 

age = int(input("Enter your age"))

while age < 0 or age > 100:
    age = int(input("Enter age between 0 and 100"))
Presence check Checks that data has been entered. Ensuring a name field is not left blank. 

name = input("Enter your name")

while name == "":
    name = input("You must enter a name")
Pattern check Ensures data follows the correct format. Checking an email contains @ and . 

email = input("Enter your email")

while "@" not in email or "." not in email:
    email = input("Enter a valid email address")
Combined checks Uses multiple validation types together. Validating a postcode format (length + pattern). 

valid = False

while not valid:
    postcode = input("Enter postcode (LL00 0LL)")

    if len(postcode) != 8:
        continue
    # Additional pattern checks here
    valid = True

Validation helps prevent incorrect or unusable data from being processed by a program.

Programming Authentication

What is Authentication?

  • Authentication is the process of ensuring that a system is secure by asking the user to complete tasks to prove they are an authorised user.
  • Authentication is used because bots can submit data in online forms.
  • Authentication can be done in several ways, including:
  • Usernames and passwords
  • CAPTCHA
  • Other methods programmers can use include:
  • Allowing users to recover passwords via email links and SMS codes
  • Encrypting data

How can authentication be programmed?

  • Authentication of usernames and passwords can be programmed using lists, databases and encryption.
  • Students must be able to program username and password lookups using a list/array.
  • To do this successfully, a 2-dimensional list/array is used.

Python Example


# Set a Boolean flag to track if the username is found
found = False

# 2D list of usernames and passwords
users = [
    ["Dave", "1"],
    ["Steve", "2"],
    ["James", "3"],
    ["Alice", "4"],
    ["Stephanie", "5"]
]

# Ask the user for their username
username = input("Enter your username: ")

# Loop through the list
for i in range(len(users)):
    if username == users[i][0]:
        found = True
        password = input("Enter your password: ")

        if password == users[i][1]:
            print("Welcome")
        else:
            print("Wrong password")
        break

# If the username was not found
if not found:
    print("User not found")
