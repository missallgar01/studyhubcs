Topic 3 Programming Languages– Revision

High-Level Programming Languages

High-level programming languages are closer to human language and are therefore easier for humans to understand. A translator is used to convert the instructions into code that the computer can understand.

High-Level Language Features

High-level languages allow programs to be written independently of the type of computer system. It is the compiler’s job to translate the code into the correct machine code for a particular computer.

Low-Level Programming Languages

Low-level programming languages include machine code and assembly language. They are difficult for humans to understand but allow direct control of the hardware.

Machine Code

Machine code is written in binary (0s and 1s) and can be processed directly by the CPU without needing to be translated. Machine code is specific to a particular processor.

Assembly Language

Assembly language uses mnemonics to represent instructions. It requires an assembler to translate the code into binary machine code.

Uses of Low-Level Languages

Low-level languages are appropriate for developing operating systems, embedded systems, and hardware device drivers.

Advantages of Low-Level Languages

Low-level languages are processor-specific and can manipulate hardware directly. They can be highly optimised, execute quickly, and use minimal memory. Each line of code represents one instruction.

Mnemonics

Mnemonics are short words used in assembly language to represent instructions, such as LOAD, ADD, STORE, and MOV.

Machine Code Instructions

Machine code instructions are made up of two parts: an operator, which is the instruction, and an operand, which is the value or memory address.

Program Translators

Program translators convert programs written in high-level or assembly languages into machine code so that they can be run on a computer.

Interpreter

An interpreter translates high-level code into machine code one instruction at a time while the program is running. It stops when an error is found, which makes debugging easier, but programs run more slowly.

Compiler

A compiler translates all the source code into machine code before the program is run. The machine code is saved as an executable file, allowing programs to run faster and be distributed without the source code.

Assembler

An assembler translates assembly language instructions into machine code so that they can be executed by the CPU.

High-Level vs Low-Level Languages

High-level languages use English-like instructions and are machine-independent, while low-level languages use mnemonics or binary, are machine-specific, and allow direct control of hardware.

Compiler vs Interpreter

A compiler translates all code at once and produces a standalone executable, while an interpreter translates code line by line each time the program runs and reports errors as they occur.
