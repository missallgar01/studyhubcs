Thinking Ahead Identifying preconditions, inputs, outputs and reusable components Preparing what is needed before starting;
Using caching;
Identifying reusable components 		+ Caching speeds up processes
− Caching can be complex or incorrect 		Calculating paint before decorating;
Preparing tools for DIY
Thinking Procedurally Breaking a problem down Identifying sub-problems;
Determining the order of events 		− Not always suitable for event-driven systems Enter marks → apply grade boundaries → print results
Thinking Logically Identifying decision points for branching or iteration Determining conditions;
Choosing actions based on outcomes 		+ Algorithm complexity can be analysed Flowcharts used to design algorithms
Thinking Concurrently More than one thing happening at the same time Identifying tasks that can run simultaneously + Faster solutions
− Hard to program
− Not suitable for all problems 		Ordering windows while building walls
