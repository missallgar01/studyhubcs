What “data types” are

Computers store everything in binary, but different kinds of information (numbers, text, True/False values) are represented in different ways. Choosing the right data type matters because it affects what operations are valid and how data is stored.

Primitive data types

  • Integer: whole numbers (including 0 and negatives); no fractional part.
  • Real / floating point: can include a fractional part; used for measurements.
  • Character: a single symbol (letters, digits, punctuation, etc.).
  • String: a sequence of characters (text; also useful for values like phone numbers where leading zeros matter).
  • Boolean: only two values: True or False.

Representing positive integers in binary

Binary is base 2. Each bit position represents a power of two. The rightmost bit is the least significant bit (LSB); the leftmost is the most significant bit (MSB).

8 (2³)4 (2²)2 (2¹)1 (2⁰)
1101

Example: 1101 = (8×1)+(4×1)+(2×0)+(1×1) = 13

Bits & bytes: 1 binary digit = 1 bit. 8 bits = 1 byte. 4 bits is sometimes called a nybble.

Binary ⇄ denary conversion

Binary → denary: multiply each bit by its place value and sum.

Denary → binary (place-value method):

  1. Find the largest power of 2 not exceeding the number.
  2. Write the place values down to 1.
  3. From MSB to LSB: write 1 if the place value fits, subtract it; otherwise write 0.

Worked example: convert 47 to binary.

32168421
101111

So 47 = 101111. Often written to a full byte with leading zeros: 00101111.

Negative numbers in binary

To represent negatives, bit patterns follow specific rules. Two key schemes are sign magnitude and two’s complement.

Sign magnitude

  • MSB is the sign: 0 = positive, 1 = negative.
  • Remaining bits are the magnitude (absolute value).
Binary (unsigned)+ (sign magnitude)− (sign magnitude)
101011010 101011011 10101101

To decode: read sign bit, drop it, convert remainder as normal, then apply the sign.

Two’s complement

  • MSB has a negative place value (for 8-bit: MSB represents −128).
  • To make a negative number: invert bits of the positive form, then add 1.

Example: represent −7 in 8-bit two’s complement.

-1286432168421
11111001

Value = (−128×1)+64+32+16+8+0+0+1 = −7

Binary addition (integers)

Core rules (including any carry-in):

  • 0+0=0
  • 0+1=1
  • 1+1=10 (write 0, carry 1)
  • 1+1+1=11 (write 1, carry 1)

Worked example: 1011 + 1110

A1011
B1110
Sum11001

Result: 11001. (Good practice: verify by converting to denary.)

Binary subtraction using two’s complement

Subtraction can be done by adding a negative number: A − B = A + (−B).

Example: 8 − 12 using 5-bit two’s complement (minimum bits needed to represent −12).

Place-168421
801000
−1210100
Sum11100

11100 = −16 + 8 + 4 = −4 (so 8 − 12 = −4).

Floating point numbers in binary

Floating point is like scientific notation: a value is stored as mantissa × 2exponent. The mantissa and exponent are typically stored in two’s complement.

The binary point for the mantissa is taken to be just after its first bit. So a mantissa like 01100100111 represents 0.1100100111.

Example layout: 11-bit mantissa and 6-bit exponent

Mantissa (11 bits) Exponent (6 bits)
01100100111 000110
How to decode: convert exponent to denary, shift the mantissa’s binary point left/right by that amount, then convert the resulting binary number to denary (including fractional place values).

Floating point example (shift right)

If the exponent is +5, move the mantissa’s binary point 5 places to the right. Then convert using powers of two including fractional values.

32168421·1/21/41/81/16
110010·0111

Value = 32 + 16 + 2 + 0.25 + 0.125 + 0.0625 = 50.4375

Floating point example (negative + shift left)

If the mantissa sign bit indicates a negative value (two’s complement), treat the mantissa as negative. If the exponent is negative, shift the binary point left.

Example concept: exponent −3 shifts the binary point 3 places left, increasing the number of leading zeros after the point.

1/21/41/81/161/321/641/1281/2561/512
000101101

This table shows the idea of converting a shifted fractional binary value using fractional place values.

Normalisation (floating point)

Normalisation makes the mantissa use as much of its available bit-width as possible (maximising precision). A common rule: ensure the mantissa begins 01 for positive values or 10 for negative values.

Example idea: shift mantissa left until it starts with 01, then decrease exponent by the number of shifts.

Mantissa (8 bits)Exponent (4 bits)
00011010 0101

After shifting mantissa left 2 places → exponent decreases by 2; resulting mantissa begins with 01.

Floating point arithmetic

Addition:

  • Make exponents equal by shifting the mantissa of the number with the smaller exponent (right shift mantissa, increase exponent).
  • Add mantissas.
  • Normalise result if needed.

Worked-structure example (same exponent after alignment):

MantissabitsExponentbits
A0010000010
B0001010010

Subtraction:

  • Align exponents (same as addition).
  • Convert the mantissa being subtracted into two’s complement.
  • Add mantissas, then normalise.

Bitwise manipulation and masks

Logical shifts

  • Logical shift left: move bits left; add zeros on the right. Equivalent to ×2k for k shifts (if no overflow).
  • Logical shift right: move bits right; add zeros on the left. Equivalent to ÷2k (integer division for integers).

Example: shift left by 3: 10010110 → 10010110000 (×8)

Masks with logic operators

A mask combines two bit patterns using a bitwise operator (AND/OR/XOR) to select, clear, toggle, or test bits.

Example AND mask:

A00101011
AND10111011
=00101011

AND outputs 1 only where both input bits are 1.

OR and XOR on the same inputs:

A00101011
OR10111011
=10111011
XOR10111011
=10010000

OR outputs 1 if either bit is 1; XOR outputs 1 only when bits differ.

Character sets for representing text

A character set maps numeric codes to characters so text can be stored and displayed. Two common character sets are ASCII and Unicode.

ASCII

  • Historically common; uses 7 bits → 128 codes.
  • Examples: ‘A’–‘Z’ map to codes 65–90; ‘a’–‘z’ map to 97–122.
  • Limited for non-English languages and many symbols.

Unicode

  • Designed to cover many languages and symbols.
  • Uses variable-length encodings (depending on format), supporting a very large character space (over a million possible code points).
