Exam command words: how to answer

STATE

What to do: Give the answer with no extra detail. Often 1 mark per correct point.
How to write: Short phrase or list item(s). No explanation needed unless asked.
Template: “[Keyword] is …” / “Two items are: … and …”
Example: “RAM stores data and instructions.”

DESCRIBE

What to do: Say what happens / what it is like in a clear sequence or key features.
How to write: 1–3 linked sentences; focus on steps, features, or a process.
Template: “First…, then…, as a result…” / “It has…, which…”
Example: “The OS uses a tree structure: folders contain subfolders, and each file has a unique path from the root.”

EXPLAIN

What to do: Give a reason and the effect (cause → because → which means). This is where marks come from.
How to write: Linked statements. Use “because”, “so”, “therefore”, “which means”. Add an example if helpful.
Template: “Explain why [Keyword] is used because [reason], which means [effect], for example [example].”
Example: “Virtual memory is used because RAM is limited, which means inactive pages are moved to disk so active programs can still run.”

DISCUSS

What to do: Consider both sides (advantages/limitations or compare alternatives) and link back to the scenario.
How to write: Multiple points for each side + conclusion/judgement. Use connectives like “however”, “whereas”, “on the other hand”.
Template:
“One advantage is…, because… which means…
However, a limitation is…, because… which means…
Overall, [choice] is more suitable for [scenario] because…”
Example: “Low-level code suits drivers (fast, hardware control), however high-level suits apps (libraries, faster development).”

Quick mark tip: If a question is 2 marks, aim for two clear points or one point with a reason. If it is 6 marks, aim for 6 distinct linked points across both sides.
