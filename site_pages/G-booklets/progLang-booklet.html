Q4. Some low-level programming languages use mnemonics.
(i) State the purpose of a mnemonic. (1)
(ii) State the type of low-level programming language that uses mnemonics. (1)
(iii) Describe one other characteristic of a low-level language. (2)
(iv) State the name of the high-level programming language translator that executes a line of code immediately after translating it. (1)
(Total for question = 5 marks)

How to answer (STATE / DESCRIBE):
  • State = short fact
  • Describe = characteristic + explanation
Mark scheme – ALL acceptable answers:
  • Mnemonic purpose:
    • Represents an instruction (1)
    • Makes instructions easier to read/write/remember (1)
  • Language: Assembly language (1)
  • Other characteristics:
    • Processor specific (1) → direct hardware manipulation (1)
    • Highly optimised (1) → efficient execution or minimal memory (1)
    • One instruction per line (1) (1)
    • Machine code written in binary → no translation needed
  • Translator: Interpreter (1)
Model full-marks answer:

A mnemonic represents a machine instruction in a readable form. Assembly language uses mnemonics. Low-level languages are processor specific, meaning they can directly control hardware. An interpreter translates and executes code line by line.

Q10. Computers
Programmers write code.
(i) Describe how an interpreter translates high-level code. (2)
(ii) Assembly language is a low-level programming language. Give two characteristics of assembly language. (2)
(iii) Name the method that records code changes, who made them and when. (1)
(Total for question = 5 marks)

Command word focus: DESCRIBE / GIVE / NAME
  • Describe = process
  • Give = list characteristics
  • Name = single term
Mark scheme – ALL acceptable answers:
  • (i) Interpreter:
    • Reads/translates one line at a time (1)
    • Executes immediately after translation (1)
    • Errors reported as soon as found (1)
    • Stops when an error is found (1)
  • (ii) Assembly language characteristics:
    • Uses mnemonics (1)
    • Hardware/chip/architecture specific (1)
    • Manipulates memory/registers directly (1)
    • One instruction maps to one CPU operation (1)
  • (iii) Audit trail / log (1)
Model full-marks answer:

An interpreter translates and executes code one line at a time, stopping when an error is found. Assembly language uses mnemonics and is hardware-specific. The method used to record changes is an audit trail.

Q14. Computers
A company is developing a new smartphone.
The smartphone has built-in devices, including a camera and a sound recorder.
The smartphone has applications, including one to edit pictures, one to translate speech to a text file and one for email.
Discuss the characteristics of high-level and low-level programming languages that make them suitable for developing software for the smartphone.
You should consider:
• the built-in devices
• the applications.
(6)
(Total for question = 6 marks)

Command word focus: DISCUSS
  • Cover both high-level and low-level languages
  • Link characteristics to devices and applications
  • Explain advantages and suitability
Mark scheme – ALL acceptable points (indicative content):
  • Built-in devices:
    • Low-level languages may be used for built-in devices
    • Low-level languages are microprocessor dependent
    • Designed to work efficiently with a single chipset
    • Can manipulate hardware directly
    • Suitable for device drivers (camera, sound recorder)
    • Can be highly optimised for speed
    • Generate smaller executables
    • Suitable for limited RAM
    • High-level languages may be used if a compiler exists for the chipset
    • High-level languages may provide libraries for device access
  • Applications:
    • High-level languages are suitable for applications
    • Applications written in high-level languages are portable
    • Development is quicker using high-level languages
    • Libraries exist for advanced tasks (image editing, speech-to-text)
    • Libraries allow applications to interface with devices
Model full-marks answer:

Low-level languages are suitable for built-in devices because they are processor-specific and can directly control hardware, allowing fast and efficient device drivers. High-level languages are suitable for applications because they are easier and faster to develop, portable, and provide libraries for tasks such as image editing, speech recognition and email.
