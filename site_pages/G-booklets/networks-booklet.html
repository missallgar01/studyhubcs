Benefits of Networking

  • Share files and data between computers
  • Share applications and software
  • Collaborate with other users
  • Share peripherals such as printers
  • Share internet connectivity
  • Central control of security
  • Central management of software updates
  • Centralised backups
  • Network monitoring and remote support
  • Communication using email and video conferencing

LAN (Local Area Network)

A LAN covers a small geographical area, such as a single building or campus. It usually uses infrastructure owned by one organisation and connects devices using switches.

WAN (Wide Area Network)

A WAN covers a large geographical area and connects multiple LANs. It often uses third-party infrastructure and connects networks using gateways.

Wired Networks

Wired networks use cables to transmit data. They offer higher speeds, greater reliability, and improved security, but cables can be difficult to maintain in large organisations.

Wireless Networks

Wireless networks use radio waves on the 2.4 GHz and 5 GHz frequency bands. They allow mobility but can suffer from interference, security issues, and slower speeds compared to wired networks.

Network Hardware

Network hardware includes wireless access points that convert signals into radio waves, switches that connect devices on a LAN, routers that connect networks, and network interface cards (NICs) that allow devices to connect to a network.

Hardware Purpose
Router Connects different networks together and routes data packets between them using IP addresses.
Switch Connects devices on a LAN and forwards data only to the intended device, improving efficiency and reducing collisions.
Wireless Access Point (WAP) Converts network signals into radio waves, allowing wireless devices to connect to a wired network.
Network Interface Card (NIC) Hardware inside a device that allows it to connect to a network and communicate with other devices.
Gateway Connects different networks that use different protocols and allows data to be transferred between them.
Modem Converts digital data from a computer into a form suitable for transmission over telephone or cable networks and back again.

IP and MAC Addresses

Every device has a MAC address that never changes and is used for communication on a LAN. Devices also have IP addresses, which can change and are used to route data across networks such as the internet.

IP Address and MAC Address (with Examples)

Address Type What it is Example Key Point
IP Address A logical address used to identify a device on a network and route data across networks such as the internet. 192.168.1.24 Can change (dynamic). Used outside the local network.
MAC Address A unique physical address built into a network interface card (NIC). Used to identify devices on a local network. 3A:5F:2C:9B:7E:41 Never changes. Used within a LAN.

Factors Affecting Wireless Network Performance

  • Walls and floors can block or weaken wireless signals.
  • Interference from other wireless networks or electronic devices.
  • Limited bandwidth available to wireless devices.
  • The number of devices connected to the network sharing the bandwidth.

Network Speed

Network speed is affected by latency, which is the delay between sending and receiving data, and bandwidth, which is the maximum amount of data that can be transmitted per second.

Formula:

Time (seconds) = File size (bits) ÷ Bandwidth (bits per second)

Network Topology

A network topology describes how devices are arranged within a network, such as star, bus, or mesh topologies.

Topology How it Works Advantages Disadvantages
Star All devices connect to a central hub or switch. All communication passes through the hub. Improved security and reliability.
If one connection fails, only one device is affected. 		If the central hub or switch fails, the entire network is affected.
Requires extra cabling and hardware.
Bus All devices connect to a single backbone cable. Terminators are used at each end of the cable. Cheap to install.
Requires less cable than a star network. 		If the backbone or terminator fails, the whole network fails.
Collisions reduce performance, especially on large networks.
Mesh Devices connect directly to each other with no central switch. Data can travel via multiple routes. Very reliable as multiple routes are available.
Data can take the fastest route. 		Expensive if wired.
Complex to set up and maintain.

The Internet

The internet is made up of backbones, network access points, and routers that forward data between networks. Each device connected to the internet is given an IP address.

DNS (Domain Name System)

DNS converts human-readable domain names and website addresses into IP addresses so data can be routed correctly.

Packet Switching

Packet switching is a method used to send data across the internet. Large pieces of data are split into smaller units called packets.

Each packet can take a different route across the network and may arrive at the destination out of order.

What information is stored in each packet?

  • Source address
  • Destination address
  • Sequence number
  • Error-checking data (checksum)

Why data is split into packets

  • Speeds up data transfer
  • Reduces reliance on one single route
  • Limits the impact of data corruption
  • Allows packets to be resent if lost

What happens at the destination?

The sequence numbers are used to place packets back into the correct order. If any packets are missing or corrupted, they are requested again.

TCP/IP Model

The TCP/IP model consists of four layers: Each layer has a specific role in sending data across a network.

Layer Example Protocols Purpose
Application HTTP, HTTPS, FTP, SMTP Selects the correct protocol depending on the application, for example viewing a web page or sending an email.
Transport TCP Splits data into packets and checks that data has been sent and received correctly between devices.
Internet IP Routes data across networks using IP addresses and adds source and destination IP addresses to packets.
Data Link Ethernet, WiFi Handles the physical transmission of data using network interface cards (NICs) and cabling or wireless signals.

Email and Web Protocols

POP3 and IMAP are used to receive emails, while SMTP is used to send emails. HTTP and HTTPS are used to request web pages, and FTP allows file transfers.

Protocol Full Name Purpose
HTTP HyperText Transfer Protocol Used to request and transfer web pages over the internet.
HTTPS HyperText Transfer Protocol Secure Secure version of HTTP that encrypts data to protect it during transmission.
FTP File Transfer Protocol Used to transfer files between computers over a network.
SMTP Simple Mail Transfer Protocol Used to send emails from a client to a mail server or between mail servers.
POP3 Post Office Protocol v3 Used to receive emails by downloading them and usually removing them from the server.
IMAP Internet Message Access Protocol Used to receive emails while keeping them on the server so they can be accessed from multiple devices.

Network Security

Network security includes access control, authentication, physical security, firewalls, encryption, and penetration testing to protect networks and data from unauthorised access.

Security Method Description
Access Control Restricts what users can access on a network, such as software, files, internet access, and the ability to install or remove programs.
Authentication Confirms the identity of a user using methods such as passwords, biometrics, or security tokens.
Physical Security Protects hardware and network infrastructure using locks, alarms, security lights, key cards, or biometric systems.
Firewalls Acts as a barrier between a local network and the internet by using rules to allow or block data packets, preventing unauthorised access.
Encryption Scrambles data using a key so that intercepted data cannot be understood unless it is decrypted.
Penetration Testing Ethical hacking used to test network security by attempting to find and fix vulnerabilities before real attackers exploit them.
