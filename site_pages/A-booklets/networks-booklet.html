1(a). Question

A company wants more customers to be able to find their website on the internet.

When customers contact the company, their computers will use a number of protocols such as TCP/IP and HTTP to make connections with other devices over the internet.

Explain what a protocol is and why they are important in network communications. [2]

  • A protocol is a set of rules for communication between devices.
  • Protocols ensure data is transferred correctly and predictably between different hardware and operating systems.

1(b). Question

Describe three different pieces of networking hardware in a client-server network apart from the clients and servers themselves. [6]

  • Router – connects different networks together and forwards data packets based on IP addresses.
  • Switch – connects multiple wired devices and forwards data only to the intended recipient using MAC addresses.
  • Network Interface Card (NIC) – allows a device to connect to a network and provides a unique MAC address.

Explain why protocol layering is used. [3]

  • Provides abstraction by hiding lower-level details

3(a)(i). Question

A LAN uses packet switching. Describe one difference between packet switching and circuit switching.

Mark scheme (one paired difference):

  • Circuit switching: dedicated resources/route are reserved for the connection; data goes along one route/stream.
  • Packet switching: network hardware/resources are shared between many connections; packets may take different routes.

3(a)(ii). Question

Explain why packet switching is more suitable for a computer network than circuit switching.

Mark scheme (any two):

  • Computer networks have many concurrent connections; packet switching shares hardware (not tied to one connection).
  • If errors occur, only missing/corrupted packets are resent (not the whole stream).
  • If a route/hardware fails, packets can be routed around failures.

3(b). Question

Zac has hired a company that will advise him on what type of LAN he should set up. Discuss how Zac could set up a peer-to-peer network and a client-server network. You should refer to:

  • how the computers in each type are connected
  • the benefits of each type
  • the drawbacks of each type
  • the suitability of each type

Indicative mark scheme points:

Peer-to-peer (P2P):

  • Computers connect as equals; each machine can share its own resources (files/printers/storage/internet).
  • Security/login is managed per-machine; little central administration needed.
  • If a computer is switched off, any resources it shares become unavailable.
  • Easy/cheap to add another machine (good for small/quick growth).

Client-server:

  • Clients connect to a server that provides network services (e.g., central storage, authentication, printing).
  • Centralised security and user management; shared resources not dependent on a staff PC being on.
  • Requires more specialist setup/admin (cost + IT skills/time).
  • If the server fails, key network functionality (including login/services) can be lost.

Suitability (apply to scenario):

  • For a small office with up to five staff, P2P can be cheaper and flexible to expand quickly.
  • Because accountancy data is sensitive, client-server can be more suitable due to stronger centralised security and controlled shared storage.

  • Protocol to be used is decided based on the application.
  • e.g. HTTPS for browser based service / SMTP/IMAP for messaging service (must be relevant; not HTTP for encrypted messaging).
  • Adds encryption.
  • Passes on to transport layer to send.
  • Gets data from transport layer when receiving.
  • Unpacks message ready for display / removes headers or other non-viewable data.
  • Decrypts message (when receiving).

4(ii). Question

ii. Explain what happens at the link layer (sometimes referred to as the “network interface layer”, “network access layer” or simply the “network layer”) of the TCP/IP stack when using this program. [2]

  • Receives (layered) data from internet layer to send.
  • MAC addresses are added to the packet/frame.
  • Passes and receives data across wireless network (to WAN / other machine).
  • Passes (layered) data back up to internet layer when receiving.

7. Question

In order to view a website, a user enters a website address into their web browser such as http://www.ocr.org.uk. The website will then be displayed onto the user’s screen.

Explain how the Domain Name System (DNS) plays a role in websites being loaded. [4]

  • The browser needs to find the IP address that corresponds to the domain name.
  • A request is sent to a DNS resolver (often provided by the ISP).
  • The resolver checks its cache; if not found, it queries other DNS servers:
    • Top Level Domain (TLD) name server
    • Authoritative name server
  • The correct IP address is returned to the client.
  • The browser then uses this IP address to connect to the web server and load the website.

Stage 8 — Q8

*A company has six solicitors working in two offices in different locations. They work with the general public and help them solve any sensitive legal issues they may be facing.

The solicitors would like to set up a computer network to allow them to work together more effectively.

Discuss the benefits and drawbacks of computer networking to the company. You should refer to:

  • Different networking methods
  • Cost implications
  • Security implications

[9]

Indicative content (MS themes):

  • Two offices → likely need a WAN connecting two LANs.
  • Client-server supports centralised security, backup, shared resources (but needs server hardware/admin → cost).
  • VPN can secure links between offices; encryption needed for sensitive case data.
  • Cloud storage enables sharing/remote access and scalable storage, but data is more “out of firm control”.
  • Benefits: shared files, collaboration, access case files from either office, shared printers/internet.
  • Security risks: increased attack surface; must protect confidential/legal data (strong access control, encryption, policies).
  • Costs: routers/switches/servers/cabling/WAP; ongoing IT support; security measures increase costs.
  • Evaluation: networking improves effectiveness but must be justified against cost and strict security needs.
