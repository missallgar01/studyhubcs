Programming Constructs

Programming constructs control the flow of execution in a program. Structured programming uses three fundamental constructs: sequence, branching, and iteration.

Sequence

Sequence means instructions are executed one after another, from top to bottom, in the order they appear.

Example: 
x = 5
y = 3
z = x + y
print(z)

The program always prints 8 because the steps are fixed.

Branching (Selection)

Branching allows different paths to be taken depending on a condition. This is implemented using IF statements.

Example: 
if mark >= 50:
  print("Pass")
else:
  print("Fail")

Iteration (Loops)

Iteration repeats a block of code. There are two main types.

Count-controlled iteration

for i in range(0, 10):
  print(i)

Condition-controlled iteration

while x <= 20:
  print("Not true")
  x = x + 1

Exam Style Question – Denary to Binary Conversion

A function, toBinary(), is needed to calculate the binary value of a denary integer between 0 and 255.

toBinary() needs to:

  • Take an integer value as a parameter
  • Divide the number by 2 repeatedly, storing a 1 if there is a remainder and a 0 if there is not
  • Combine the remainder values (first to last, running right to left) to create the binary number
  • Return the binary number

For example, to convert 25 to a binary number the steps are as follows: 

25 / 2 = 12   remainder 1
12 / 2 = 6    remainder 0
6 / 2 = 3     remainder 0
3 / 2 = 1     remainder 1
1 / 2 = 0     remainder 1

Return value = 11001

Write the function toBinary(). You should write your function using pseudocode or program code. [6]

Mark Scheme (1 mark per bullet)

  • Function header taking a parameter
  • Appropriate loop (e.g. until value is 0)
  • Division by 2 and finding the remainder (e.g. MOD)
  • Correctly adding 1 or 0
  • Appending digits correctly / final string reversed
  • Reducing the value inside the loop
  • Returning the calculated value

Pseudocode

function toBinary(denary)
  binaryValue = ""

  while denary > 0
    temp = denary MOD 2

    if temp == 1 then
      binaryValue = "1" + binaryValue
    else
      binaryValue = "0" + binaryValue
    endif

    denary = denary DIV 2
  endwhile

  return binaryValue
endfunction

Python

def toBinary(denary):
    binaryValue = ""

    while denary > 0:
        temp = denary % 2

        if temp == 1:
            binaryValue = "1" + binaryValue
        else:
            binaryValue = "0" + binaryValue

        denary = denary // 2

    return binaryValue

Recursion

Recursion occurs when a subroutine calls itself. It must always include a base case to stop infinite calls.

Example: Factorial 
function factorial(n):
  if n == 0 or n == 1:
    return 1
  else:
    return n * factorial(n - 1)
end function

Recursion vs Iteration

Recursion Iteration
Often shorter and clearer Usually more memory efficient
Uses call stack Uses loops
Risk of stack overflow No stack overflow risk

Call Stack and Stack Frames

Each recursive call creates a stack frame storing parameters, local variables, and return addresses. Frames are removed as the recursion unwinds.

Example:
Calculating factorial(4) creates stack frames for 4 → 3 → 2 → 1 before returning values.

Scope of Variables

Scope is the section of code in which a variable can be accessed. A local variable within a subroutine can only be accessed within that subroutine, whereas global variables can be accessed across the whole program.

Local Variables

  • Only work inside the subroutine where they are created.
  • Deleted when the subroutine finishes.
  • Help keep subroutines independent and organised.

Global Variables

  • Can be used anywhere in the program.
  • Helpful for shared values.
  • Risk of accidental changes.
  • Stay in memory until the program ends, using more space.

Local Variables

Local variables are defined inside a subroutine and can only be accessed within that subroutine. 

function add():
  total = 0   // local variable
  total = total + 5
end function

Global Variables

Global variables are accessible throughout the entire program. They persist for the program’s lifetime. 

score = 0   // global variable

function increase():
  score = score + 1
end function

Using many global variables is discouraged because they can be overwritten accidentally and consume memory longer.

Modularity

Modularity breaks a large program into smaller, self-contained modules.

  • Easier to understand
  • Easier to test and maintain
  • Encourages reuse

Top-Down Design (Stepwise Refinement)

A problem is repeatedly broken into smaller sub-problems until each can be solved independently.

Example:
A booking system is divided into: login → search → reserve → confirm.

Exam Style Question – String Validation

The function validateAnswer takes in the random letters as an array of letters and the player’s answer as a string.

It checks whether the word entered only contains letters from the 10 random letters, with each letter being used only once. (At this stage the program does not check if the answer is a real word.)

The function returns a score out of 10 for a valid word, or 0 for an invalid word.

Example

  • If the random letters are OPXCMURETN
  • The word COMPUTER returns 8
  • The word POST returns 0 (no S)
  • The word RETURN returns 0 (only one R available)

Complete the function validateAnswer(answer, randomLetters[]). [6]

Mark Scheme

  • Traverses every letter of the answer
  • Traverses the randomLetters array
  • Checks each answer letter against random letters
  • Returns 0 if a letter does not occur
  • Returns 0 if a letter occurs too many times
  • Returns answer length for a valid word

Solution (Pseudocode)

function validateAnswer(answer, randomLetters)
  valid = True
  score = 0

  lengthAnswer = len(answer)
  lengthRandom = len(randomLetters)
  index = 0

  while (valid == True) and (index < lengthAnswer)
    numChar = 0
    currentLetter = answer[index:index+1]

    for i = 0 to lengthRandom - 1
      if currentLetter == randomLetters[i] then
        numChar = numChar + 1
      endif
    next i

    if numChar == 0 then
      valid = False
    endif

    index = index + 1
  endwhile

  if valid == True then
    score = lengthAnswer
  endif

  return score
endfunction

Example call: 

print(validateAnswer("POST", "OPXCMURETN"))

Working with Text Files

Opening Files

Before reading from, writing to, appending to, or modifying a file, the file must first be opened. 

open(filename, access_mode)

There are a range of access modes depending on what we want to do to the file.

Access Mode Description
r Opens a file for reading only
w Opens a file for writing only. Creates a new file if one does not exist. Overwrites the file if it already exists.
a Appends to the end of a file. Creates a new file if one does not exist.

Reading Text Files

read – reads the whole file into a single string

f = open("file.txt", "r")
print(f.read())
f.close()

readline – reads one line at a time

f = open("file.txt", "r")
print(f.readline())
print(f.readline())
print(f.readline())
f.close()

readlines – reads the whole file into a list

f = open("file.txt", "r")
print(f.readlines())
f.close()

Writing Text Files

Writing single lines

file = open("days.txt", "w")
file.write("Monday\n")

Writing from a list

say = ["How\n", "are\n", "you\n"]
file = open("say.txt", "w")
file.writelines(say)
file.close()

Programming Standards

  • No function should be longer than a single page of code. This reduces complexity and improves readability.
  • Variable identifiers must follow a standard naming convention. This helps others understand the code, reduces duplication, and makes maintenance easier.
  • Each function must have a single entry point. This reduces complexity and makes bugs easier to locate.
  • Variables should not be set up outside the scope of a function. This limits where bugs may occur and prevents problems spreading across multiple modules.
  • Code must be correctly indented.
  • Global variables should use upper case with underscores, e.g. UPPER_CASE_WITH_UNDERSCORES.
  • In Python, function names should be lowercase.
  • Comment code appropriately. Avoid line-by-line comments as they reduce readability.

Test Data

Programs must be tested with a range of different input data to ensure they work as expected under all conditions.

Input data should be checked to ensure values are:

  • Within a certain range
  • In the correct format
  • The correct length
  • The correct data type (e.g. integer, float, string)

Programs are tested using normal, erroneous, and boundary data.

Normal Data

Data that is expected to be entered. For example, the age of a secondary school pupil would typically be an integer between 11 and 19.

Erroneous Data

Data that is clearly incorrect. For example, entering 40 as the age of a school pupil. The program should identify this as invalid and respond sensibly without crashing.

Boundary Data

Data at the extreme limits of acceptable values. For example, ages of 10, 11, 19, or 20.

Debugging

Syntax Errors

Errors in the code that prevent the program from running. These include missing brackets, spelling mistakes, and other typographical errors.

Runtime Errors

Errors that occur while the program is running. For example, attempting to access a memory location that does not exist, such as an invalid array index.

Logical Errors

The program runs to completion but produces incorrect output. These errors are often caused by incorrect calculations or faulty logic.

Question:
Describe how low-level programming differs from procedural programming. [6]

Mark scheme points (any suitable):

  • Low-level programming is usually carried out in assembly language.
  • Assembly instructions correspond closely to machine code executed on the CPU.
  • Each assembly instruction contains an opcode and operands (e.g. addresses, values, registers).
  • Low-level programming requires careful consideration of CPU architecture / how it works.
  • Procedural programming uses high-level instructions (e.g. print), which are translated into multiple low-level instructions by a compiler.
  • High-level control structures (e.g. while / for) have clearer syntax and are generally easier to read and write.
