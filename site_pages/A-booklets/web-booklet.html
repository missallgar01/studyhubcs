Software & Operating Systems – Notes

HTML — Structure of Web Pages

HTML defines the structure and order of content on a webpage. It tells the browser what each piece of content is, not how it should look.

Example: A simple HTML page 
<html>
  <head>
    <title>My Website</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This is my webpage</p>
  </body>
</html>

HTML Elements — Images, Links, Lists

Images use the <img> tag:

<img src="logo.png" width="200" height="80">

Links use the anchor tag:

<a href="https://www.example.com">Visit Example</a>

Lists:

<ul>
  <li>Computer Science</li>
  <li>Maths</li>
</ul>

CSS — Styling Web Pages

CSS controls the appearance of HTML elements. It separates presentation from content.

Example: External CSS file 
body {
  background-color: white;
  font-family: Arial;
}

h1 {
  color: blue;
}

This CSS will apply to every page that links to the stylesheet.

Classes and IDs (CSS Selectors)

Classes can be reused. IDs must be unique.

Example: 
/* CSS */
.highlight {
  background-color: yellow;
}

#main {
  border: 2px solid black;
}
        
<div id="main">
  <p class="highlight">Important text</p>
</div>

JavaScript — Adding Interactivity

JavaScript is interpreted by the browser and allows pages to respond to user actions.

Example: Change text when a button is clicked 
<button onclick="changeText()">Click me</button>
<p id="msg">Hello</p>

<script>
function changeText() {
  document.getElementById("msg").innerHTML = "Goodbye";
}
</script>

Why JavaScript is Useful

  • Checks form input before sending data to the server
  • Reduces server load
  • Improves user experience
Example: Simple validation 
if (password.length < 8) {
  alert("Password too short");
}

Search Engine Indexing

Search engines use web crawlers to collect information about pages.

Example:

A crawler visits Page A, follows links to Pages B and C, extracts keywords like “computer science”, and stores them in an index.

PageRank Algorithm (Worked Concept Example)

PageRank measures importance based on incoming links.

Example:

If Page C is linked by two pages, and one of those pages is already highly ranked, Page C’s rank will increase more than if both links came from low-ranked pages. 

PR(C) = (1 − 0.85) + 0.85 × (PR(B)/2 + PR(D)/3)

Server-Side Processing — Example

Data is sent to a server, processed, and a response is returned.

Example:

A login form sends a username and password to a server. The server checks the database and responds with success or failure.

Client-Side Processing — Example

Processing happens on the user’s device.

Example:

A form checks that an email address contains “@” using JavaScript before allowing submission.

Exam Tip

Always link examples to advantages:

  • Client-side → fast response, reduced server load
  • Server-side → security, large processing tasks

1(b). Question

Discuss why search engine indexing and page ranking will be important in achieving this.

Include:

  • what indexing means and what information is collected
  • what PageRank means and factors affecting it
  • ways to improve PageRank
  • Indexing: web crawlers scan pages, record keywords/meta data, and store them in a search engine index.
  • PageRank: ranks pages based on number and quality of incoming links.
  • Factors: links from high-ranking pages increase rank more.
  • Improvements: good keywords, headings, meta tags, quality inbound links.
  • Damping factor: probability a user stops clicking links.

QP Q2

An online shop uses a website to sell its products. Part of the HTML code used for a login form is shown. The function errorCheck() runs after a password is entered and checks password length.

Write JavaScript code for errorCheck which:

  • checks password length is between 8 and 12 characters (inclusive)
  • displays "Password Length Error" in the HTML label with id errorText if invalid

Mark scheme possible solution:

function errorCheck() {
  var pass = document.getElementById("Password").value;
  if (pass.length > 12 || pass.length < 8) {
    document.getElementById("errorText").innerHTML = "Password Length Error";
  }
}

4(b). Question

Complete the JavaScript function bookingfee(numtickets, country). The booking fee is £2.99 per ticket, plus an additional £4.99 if the customer is not from the UK. 

function bookingfee(numtickets, country) {
  var nonUKprice = 4.99;
  var perTicketPrice = 2.99;
  var total = 0;

  if (country != "UK") {
    total = total + nonUKprice;
  }

  total = total + (numtickets * perTicketPrice);
  return total;
}

4(c). Question

(i) Explain why the server will calculate the booking fee again.

(ii) Give one advantage of client-side processing.

(i)

  • Client-side values can be altered/tampered with or scripting disabled; server-side ensures the final fee is correct and secure.

(ii)

  • Faster response for the user (no round-trip to server), or reduces server load/bandwidth.

4(d). Question

(i) Describe what is meant by indexing when used by search engines.

(ii) Define what is meant by a damping factor.

(iii) Give two other factors that affect the output of the PageRank algorithm.

(i) Indexing

  • Crawler/spider visits pages (from a list or by following links), records page info (text/meta tags/keywords), stores it in an index, then follows links onwards (robots.txt can guide crawling).

(ii) Damping factor

  • A value between 0 and 1 representing the probability a user stops following links / limits infinite link-following.

(iii) Other PageRank factors

  • Number of incoming links to the page.
  • PageRank (importance) of the linking pages.

4(e). Question

Describe the purpose of HTML and CSS in a website used to sell concert tickets.

  • HTML: defines the structure/content of the page using tags (headings, forms, images, links etc.).
  • CSS: controls presentation/appearance (layout, colours, fonts) using selectors such as classes and IDs; can be external/inline, and more specific rules override less specific ones.
