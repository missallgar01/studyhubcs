Software & Applications

System Software vs Application Software

System software enables the computer to operate, controls hardware and provides a platform for applications (for example, the operating system, utilities and device drivers).

Application software consists of programs that perform tasks for the user, such as writing documents, browsing the web or editing images.

Types of Application Software

Type Purpose Examples
Word Processor Create and edit text documents. Word, Google Docs
Spreadsheet Analyse and process numerical data. Excel, Google Sheets
Database Store, query and manage data. Access, MySQL
Presentation Software Create slideshow-style presentations. PowerPoint, Keynote
Graphics / Image Editor Manipulate and create graphics. Photoshop, GIMP
Web Browser Access and view web pages. Chrome, Firefox, Edge
Video Editor Edit and create video content. Premiere Pro, iMovie
IDE Write, run and debug code. VS Code, PyCharm

Utility Software

Utility software helps to configure, optimise and maintain a computer system. It supports the operating system rather than directly performing user tasks.

Utility Purpose Example Use
Disk Defragmenter Rearranges files on disk into contiguous blocks. Speeds up file access on HDDs.
Backup Utility Automatically copies data to another storage location. Nightly backups of coursework to an external drive or cloud.
Antivirus Detects and removes malware. Scans downloads and attached files.
Firewall Monitors and controls network traffic. Blocks unauthorised access to/from the network.
Compression Reduces file sizes. Compresses large video or data files to save space and bandwidth.

Open Source vs Closed Source Software

Open source software makes the source code available so that users can view, modify and share it. Closed source (proprietary) software keeps the source code secret and is usually licensed for a fee.

Type Benefits to User Drawbacks to User
Open Source Often free; transparent; can be customised; large community support. May be less polished; user support can be inconsistent; compatibility issues.
Closed Source Professional support; polished interface; thoroughly tested. Licence cost; little or no access to source; limited customisation.

Functions of an Operating System

  • Provides a user interface (GUI, CLI, etc.).
  • Manages memory (allocates and deallocates RAM).
  • Manages the processor (scheduling tasks on the CPU).
  • Manages devices via device drivers.
  • Manages files on secondary storage.
  • Provides security and access control.
  • Provides and coordinates utility software.

Types of Operating System

  • Single-user – only one user can log in at a time.
  • Multi-user – many users share a powerful computer (e.g. mainframe).
  • Multitasking – appears to run many programs at once by quickly switching between them.
  • Distributed – processing is spread over multiple computers working together.
  • Embedded – built into a device with a dedicated role (e.g. washing machine, router).
  • Real-time – guarantees responses within a strict time limit for safety-critical tasks.

Memory Management, Paging, Segmentation & Virtual Memory

Memory management controls how RAM is allocated to processes, ensuring they do not interfere with each other and that memory is used efficiently.

  • Paging – memory is split into fixed-size pages; processes are split into pages which can be loaded into any free frame.
  • Segmentation – memory is split into variable-sized segments representing logical divisions such as code, data and stack.
  • Virtual memory – uses backing store (secondary storage) as if it were extra RAM, swapping pages in and out when needed.

Disk thrashing occurs if the system spends more time swapping pages in and out of virtual memory than doing useful work, often causing serious slow-down.

BIOS, Interrupts, Scheduling & Virtual Machines

  • BIOS (Basic Input/Output System) is firmware stored on the motherboard. On startup it performs POST (Power-On Self-Test) and then loads the operating system.
  • Interrupt – a signal that temporarily halts the CPU so that a more urgent task (handled by an Interrupt Service Routine) can run.
  • Scheduling – the OS decides which process runs next. Algorithms include Round Robin, First Come First Served, and Shortest Job First.
  • Virtual machine – a software emulation of a computer, running its own OS in a sandboxed environment on top of a host system.

Check Your Understanding – Questions

1. System software vs application software

(a) Define system software.
(b) Define application software.

(a) System software enables the computer to operate, managing hardware and providing a platform for application programs.

(b) Application software is used by the user to carry out specific tasks, such as word processing, browsing the web or editing images.

2. Utility software in everyday use

A student stores coursework and downloads videos. Describe the purpose of each of these utilities in that context:

  • Compression software
  • Antivirus software
  • Backup utility

Compression: Reduces file sizes so large videos and projects take up less space and transfer more quickly.

Antivirus: Scans files for malware before they are opened, protecting coursework and the system.

Backup utility: Automatically copies important files, such as coursework, to another location so they can be restored if lost.

3. Classifying software

Classify each item as general-purpose, special-purpose, bespoke or utility software:

  • Graphics editor used to crop product photos
  • Custom-built ordering system for a single shop
  • Firewall running on a web server
  • Online payment verification module
  • Graphics editor – general-purpose application software.
  • Ordering system – bespoke software.
  • Firewall – utility software.
  • Payment verification – special-purpose software.

4. Device drivers

(a) What is a device driver?
(b) Why does a smart meter for electricity use need a driver on the user’s computer?

(a) A device driver is software that controls a particular hardware device and provides a standard interface to the operating system.

(b) The OS needs the driver so it can communicate with the smart meter and receive readings in a format it understands.

5. Off-the-shelf vs bespoke

Give three reasons why a company may choose an off-the-shelf package rather than having bespoke software written.

  • Lower cost – development cost is shared between many customers.
  • Available immediately – no long development time.
  • Usually well-tested and stable – many bugs already found and fixed.

6. Functions of an operating system

State three different functions of an operating system.

  • Provides a user interface.
  • Manages memory and processor time.
  • Manages input/output devices and file storage.
  • Provides security and user accounts.

(Any three valid functions gain credit.)

7. Real-time operating systems

(a) Define a real-time operating system.
(b) Explain why a real-time OS is suitable for an intensive care monitoring system.

(a) A real-time OS guarantees that it will respond to inputs within a fixed maximum time.

(b) In intensive care, alarms and changes in vital signs must be detected and acted on immediately. Missing or delaying an alarm could be life-threatening, so guaranteed response times are essential.

8. Virtual memory & disk thrashing

(a) What is virtual memory and why is it needed?
(b) What is disk thrashing and why is it a problem?

(a) Virtual memory uses part of secondary storage as if it were RAM, allowing larger programs to run when physical memory is insufficient.

(b) Disk thrashing occurs when the computer spends most of its time swapping pages between RAM and disk instead of doing real work, causing the system to slow down dramatically or appear to freeze.

9. Paging vs segmentation

(a) Describe paging.
(b) Describe segmentation.
(c) Give one similarity and one difference.

(a) Paging divides memory into fixed-size blocks called pages and splits processes into pages that can be loaded into any free frame.

(b) Segmentation divides memory into variable-size logical segments, such as code, data and stack.

Similarity: Both are ways of partitioning memory and can be used with virtual memory.
Difference: Pages are fixed-size physical blocks; segments are variable-size logical blocks.

10. Round Robin scheduling

Describe how the Round Robin scheduling algorithm works and give one advantage and one disadvantage.

How it works: Each ready process is given a fixed time slice of CPU time in turn. When its time slice ends, it goes to the back of the queue and the next process runs.

Advantage: Fair – each process gets regular CPU time; good for time-sharing systems.

Disadvantage: If the time slice is too small there is lots of context switching, which reduces efficiency. Long jobs may take a long time to complete.
