FAST FOUR STARTER

1. Explain

[2]

Explain one way an audit trail helps programmers create robust software.

Answer: • An audit trail improves accountability because it keeps track of who makes changes / when changes are made
• An audit trail makes going back to an earlier version easier because changes are tracked

2. Explain

[2]

A company stores statistics about its business on a server. Explain one type of access to the statistics file a student on work experience at the company should be given.

Answer: • An explanation to include one mark for a type of access and one mark for a linked reason, such as:
• To recommend read only because the student cannot accidentally change/delete the data / because the student cannot share the data
• No access because the student cannot see any sensitive/irrelevant information
