1. State/Identify

[2]

State two benefits of subprograms.

Answer: • The subprogram code only has to be written once / saves development time
• The subprogram can be called many times (reusability)
• The subprogram only has to be debugged once
• It is easier to locate and debug errors
• Hides the details of how a function/procedure/code works (abstraction)
• Can be used by programmers who don’t have the knowledge to write them
• Can be grouped into libraries / can be shared with other programmers
• Makes programs easier to read / understand
• Reduces the overall size of code
• Problem can be decomposed into smaller sub-problems
• Allows a team to work on a project

4. Define

[2]

Programmers use abstraction to model the real world. Define the term 'abstraction'.

Answer: • A definition to include:
• The process of removing or hiding unnecessary details so that only the important points remain
• Programmers can focus only on the important details of a problem because abstraction allows them to ignore any detail that is not relevant
