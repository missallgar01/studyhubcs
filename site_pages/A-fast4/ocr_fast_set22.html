FAST FOUR STARTER

1. Identify

[1]

One benefit of modular programming

Answer: • Breaks a large program into smaller, self‑contained modules that are easier to understand
• Allows different programmers to work on separate modules in parallel
• Encourages code reuse, as modules can be used in other programs
• Makes testing and debugging easier because faults can be isolated to individual modules

2. Identify

[1]

One drawback of automation.

Answer: • Over‑reliance on automated processes can mean human operators lose skills or situational awareness
• Automated systems can propagate errors very quickly if they are misconfigured
• Introducing automation often requires significant up‑front cost and setup time
• Jobs and roles may be reduced or eliminated, leading to social and economic concerns

3. Explain

[1]

One drawback of automation in testing.

Answer: • Automated tests only check the cases that have been explicitly programmed, so unexpected edge cases may be missed
• Setting up and maintaining test frameworks and scripts can be time‑consuming and costly
• Fragile tests can frequently break when the user interface or system design changes
• May lead to over‑reliance on automated checks and reduced exploratory testing by humans

4. Explain

[1]

Why virtual memory is needed

Answer: • Physical RAM is limited, but programs may require more memory than is physically available
• Allows many programs to appear to have their own large, contiguous address space
• Enables multitasking without each process having to fit entirely in RAM
• Provides memory protection so faulty processes are less likely to corrupt others
