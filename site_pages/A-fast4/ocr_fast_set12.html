1. State

[1]

One drawback of modular programming.

Answer: • Requires additional planning to design clear module interfaces
• Too many small modules can make the system structure more complex to understand
• Communication between modules (via parameters or messages) introduces some overhead
• Changes to shared interfaces can impact many modules at once

2. Explain

[1]

How merge sort works.

Answer: • Uses a divide‑and‑conquer strategy to sort a list
• Recursively splits the list into smaller sublists until each sublist contains a single element
• Merges pairs of sorted sublists into larger sorted lists by repeatedly selecting the smallest head element
• Continues merging until one fully sorted list remains
• Has predictable time complexity of O(n log n) and is a stable sort

3. Describe

[1]

One disadvantage of open-source software.

Answer: • May have less formal support; users may need to rely on community help
• Documentation quality can be inconsistent between projects
• Fragmentation can occur if many different versions or forks are created
• Some projects may be abandoned if maintainers lose interest

4. Explain

[1]

What the Control Unit does.

Answer: • Fetches instructions from memory and decodes their opcodes
• Generates control signals to coordinate the ALU, registers, buses and memory
• Manages the timing and sequencing of operations within the CPU
• Ensures the steps of the fetch‑decode‑execute cycle occur in the correct order
