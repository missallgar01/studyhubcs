1. Describe

[1]

How interrupts are handled by the CPU

Answer: • The CPU finishes executing the current instruction before responding to the interrupt
• The current state, including the Program Counter and key registers, is saved (often on the stack)
• The CPU jumps to the address of the appropriate Interrupt Service Routine (ISR)
• The ISR runs to handle the event, such as servicing an I/O device or timer
• When the ISR completes, the saved state is restored and the interrupted program resumes from where it left off

One drawback of open standards.

Answer: • Can reduce incentives for companies to innovate proprietary features
• Different implementations of the same standard may not behave identically, causing compatibility issues
• Changes to the standard require consensus, so evolution can be slow
• Harder to enforce strict quality control when anyone can implement the standard

3. Describe

[1]

How bubble sort works.

Answer: • Repeatedly passes through the list, comparing adjacent elements
• Swaps neighbouring elements if they are in the wrong order
• After each full pass, the largest unsorted element has ‘bubbled’ to its correct position at the end of the list
• Continues making passes until a pass occurs with no swaps, meaning the list is sorted
• Has a worst‑case time complexity of O(n²) and is simple but inefficient for large lists

4. State

[1]

One function of BIOS

Answer: • Performs the Power‑On Self‑Test (POST) to check CPU, memory and basic hardware
• Initialises and configures hardware components during startup
• Locates and loads the bootloader or operating system from storage into memory
• Provides low‑level routines for accessing hardware before the OS is loaded
