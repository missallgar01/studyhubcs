1. Identify

[1]

One drawback of modular software.

Answer: • Designing and agreeing clear interfaces between modules requires extra time and planning
• Too many small modules can increase the complexity of the overall system structure
• Changes to shared interfaces may require coordinated updates across multiple modules
• Communication between modules (function calls, messages) introduces a small performance overhead

2. Describe

[1]

How bubble sort works

Answer: • Repeatedly passes through the list, comparing adjacent elements
• Swaps neighbouring elements if they are in the wrong order
• After each full pass, the largest unsorted element has ‘bubbled’ to its correct position at the end of the list
• Continues making passes until a pass occurs with no swaps, meaning the list is sorted
• Has a worst‑case time complexity of O(n²) and is simple but inefficient for large lists

3. Describe

[1]

What a stack is used for.

Answer: • Stores return addresses when subroutines and functions are called
• Holds local variables and parameters for active procedures
• Keeps track of nested and recursive function calls via stack frames
• Provides a last‑in, first‑out (LIFO) structure for temporary data storage
• Supports evaluation of arithmetic expressions and undo operations

4. Describe

[1]

One benefit of Big O notation.

Answer: • Provides a way to describe how an algorithm’s running time or memory usage grows with input size
• Allows comparisons between algorithms independently of hardware or implementation details
• Helps programmers choose more efficient algorithms for large inputs
• Highlights whether an algorithm will scale well as data sizes increase
