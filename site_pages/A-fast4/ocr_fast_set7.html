1. Explain

[1]

How merge sort works

Answer: • Uses a divide‑and‑conquer strategy to sort a list
• Recursively splits the list into smaller sublists until each sublist contains a single element
• Merges pairs of sorted sublists into larger sorted lists by repeatedly selecting the smallest head element
• Continues merging until one fully sorted list remains
• Has predictable time complexity of O(n log n) and is a stable sort

2. Describe

[1]

One disadvantage of real-time systems.

Answer: • Need rigorous testing and verification to ensure timing constraints are always met
• Typically cost more to design, implement and maintain than non‑real‑time systems
• Hardware and software choices are constrained by the need for predictability
• Failure to respond in time can cause physical damage, safety hazards or financial loss

3. Describe

[1]

One drawback of binary representation.

Answer: • Large binary numbers are difficult for humans to read and interpret directly
• Requires conversion to decimal or hexadecimal for easier understanding
• Representing some fractional values exactly is not possible with a fixed number of bits
• Bit‑level errors can drastically change the represented value

4. Explain

[1]

How paging and segmentation differ.

Answer: • Paging divides memory into fixed‑size blocks (pages), whereas segmentation uses variable‑size blocks (segments)
• Pages are based on physical memory management; segments reflect logical program structures such as functions or data areas
• Paging typically avoids external fragmentation but may have internal fragmentation
• Segmentation can suffer from external fragmentation as segments vary in size
• Segmentation makes it easier to apply different protection or access rights to different logical units
