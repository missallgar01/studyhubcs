1. Describe

[1]

One drawback of high-level languages.

Answer: • Programs may run more slowly than equivalent low‑level or assembly code due to abstraction overhead
• The programmer has less direct control over hardware resources such as memory and registers
• Behaviour can depend on compiler or runtime implementation details
• Some hardware‑specific optimisations are harder or impossible to express

2. State

[1]

The purpose of the Program Counter (PC).

Answer: • Holds the memory address of the next instruction to be fetched
• Automatically increments after each fetch to keep program flow sequential
• Can be changed by branch/jump instructions to alter the flow of control
• Allows the CPU to track where it is in the current program

4. Describe

[1]

One disadvantage of Big O notation.

Answer: • Ignores constant factors and lower‑order terms, so it may not reflect actual performance on small inputs
• Does not account for practical considerations such as cache behaviour or implementation details
• Can give a misleading impression that algorithms with the same Big O are equally efficient
• Focuses only on worst‑case or general growth, not on typical real‑world usage patterns
