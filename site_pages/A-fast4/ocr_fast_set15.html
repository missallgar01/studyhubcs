1. Explain

[1]

One drawback of caching.

Answer: • Cache coherence and consistency can be difficult to maintain in multi‑core systems
• If a program has poor locality, the cache may provide little benefit but still consume resources
• Increases hardware complexity and cost
• Out‑of‑date cached data can cause incorrect behaviour if not managed correctly

2. Explain

[1]

The benefit of high-level languages.

Answer: • Provide constructs that are closer to human language and problem domains, making programs easier to write and understand
• Are generally portable across different hardware platforms once recompiled
• Include features such as data structures, libraries and automatic memory management that increase productivity
• Reduce the likelihood of low‑level errors compared with assembly language

3. Describe

[1]

How interrupts are handled by the CPU.

Answer: • The CPU finishes executing the current instruction before responding to the interrupt
• The current state, including the Program Counter and key registers, is saved (often on the stack)
• The CPU jumps to the address of the appropriate Interrupt Service Routine (ISR)
• The ISR runs to handle the event, such as servicing an I/O device or timer
• When the ISR completes, the saved state is restored and the interrupted program resumes from where it left off

4. Explain

[1]

One drawback of Unicode.

Answer: • Uses more storage and bandwidth than some older single‑byte encodings for texts using only ASCII characters
• Multiple encoding forms (UTF‑8, UTF‑16, UTF‑32) can cause compatibility and conversion issues
• Software must handle variable‑length encodings correctly, which adds complexity
• Legacy systems may not fully support Unicode, causing interoperability problems
