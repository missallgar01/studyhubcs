Jack plays rugby at his local rugby club. He wants to create a program to store the results of each rugby match and the names of the try scorers.

a) Define what is meant by abstraction. [2]
b) Give one example of how abstraction could be used when developing this program. [1]

a) Abstraction is removing unnecessary detail from a problem so that you can focus on the important parts that need solving. This reduces complexity and makes the problem easier to solve.

b) For example, the program could store the match result and the names of the try scorers, while ignoring details such as the time the try was scored, the player’s shirt number, or the venue.

Programs can have syntax errors and runtime errors.
(i) Define the term ‘syntax error’. (1)
(ii) Runtime errors happen when a program is executing. Explain a runtime error. (2)
(Total for question = 3 marks)

Command word focus: DEFINE / EXPLAIN
  • Define = what it is
  • Explain = what causes it and what happens
Mark scheme – ALL acceptable answers:
  • (i) Syntax error:
    • Error in the rules/grammar of the programming language (1)
    • Prevents the program from running (1)
  • (ii) Runtime error:
    • Error occurs while the program is executing (1)
    • Program crashes or stops unexpectedly (1)
    • Caused by invalid input, division by zero, missing file, etc. (1)
Model full-marks answer:

A syntax error is a mistake in the rules of the programming language that prevents the program from running. A runtime error occurs while the program is running, for example when invalid data causes the program to crash.

A group of students are working together on a single maze game.
The arrow keys control the character. When the character reaches the end of the maze without touching a wall, a happy sound is played. The game also displays a score.

Discuss the use of decomposition and abstraction in developing this game.
Your answer should include:
• a definition of each term
• the benefits each brings to the group of students
• an example of where each could appear in the program code.

(Total for question = 6 marks)

Command word focus: DISCUSS
  • Define both terms
  • Explain benefits
  • Give concrete code examples
Mark scheme – ALL acceptable points:
  • Decomposition:
    • Breaking a problem into smaller parts
    • Allows tasks to be shared among students
    • Examples: movement control, collision detection, scoring, sound
  • Abstraction:
    • Removing unnecessary detail
    • Makes code easier to understand and manage
    • Examples: using variables for score, functions for movement
Model full-marks answer:

Decomposition involves breaking the game into smaller parts such as movement, scoring and sound, allowing different students to work on each part. Abstraction removes unnecessary detail, for example by using functions to control movement without showing how the movement works internally.
