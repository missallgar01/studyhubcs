Computers manipulate binary patterns. Patterns can represent signed or unsigned integers.

(i) Convert the denary number 57 to 8-bit binary. (2)

(ii) Convert the binary number 0010 1101 to hexadecimal. (2)

(iii) Describe the process of converting a binary number to two's complement. (2)

(iv) Explain what has happened as a result of adding these two 8-bit binary numbers. (2)

(Total for question = 8 marks)

Command word focus:
  • Convert – show the correct final value
  • Describe – explain the steps in order
  • Explain – give a cause and an effect
Mark scheme – ALL acceptable answers:
  • (i) Award one mark for each correct nibble:
    • 0011 (1)
    • 1001 (1)
  • (ii) Award one mark for each digit in the correct location:
    • 2D (2)
  • (iii) A linked description such as:
    • Flip all the bits (1) and add one (1)
    • Copy/keep all the 0s from the right/LSB up to and including the first 1, then flip the remaining bits (2)
    • Flip all the bits from the left/MSB up to but not including the 1 nearest the right/LSB (2)
  • (iv) A linked explanation such as:
    • There are not enough bits to store the result (1)
    • An overflow occurs (1) because the result is greater than 255 (1)
    • An overflow occurs because 1 + 1 = 0 with a carry (1)
Model full-marks answer:

(i) 57 in 8-bit binary is 00111001.
(ii) 0010 1101 in hexadecimal is 2D.
(iii) To convert to two’s complement, all the bits are flipped and one is added.
(iv) The result causes an overflow because there are not enough bits to store a value greater than 255.
