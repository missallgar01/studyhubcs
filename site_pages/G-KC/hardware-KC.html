Fetching an instruction from memory

Describe the role of the control unit, the control bus, the data bus and the address bus when fetching an instruction from memory.

(4 marks)

A typical answer would include, for example:

  • The control unit places the address of the next instruction in the MAR and coordinates the fetch.
  • The address bus carries the address from the MAR to main memory.
  • The control bus sends a read signal to memory to request the instruction.
  • The data bus carries the instruction from memory back into the MDR and then to the CPU for decoding and execution.

CPU and main memory working together

Describe how the CPU and main memory work together to run programs.

(4 marks)

Fill in the missing words in this sentence:

INSTRUCTIONS    WRITTEN    DATA    FETCHED    DECODED

Program ________________ and ________ are stored in main memory are ____________ one at a time to the CPU, where they are ____________ and executed.

Completed sentence:
Program INSTRUCTIONS and DATA are stored in main memory are FETCHED one at a time to the CPU, where they are DECODED and executed.

Results of operations carried out in the CPU are WRITTEN to memory.

Data bus – why bi-directional?

Describe one reason the data bus needs to be bi-directional.

(1–2 marks)

Example answer:

  • The CPU needs to be able to read program instructions and data from memory and write the results of operations back to memory.

Clock speed

State the reason why a higher clock speed is desirable.

(1 mark)

Answer: More instructions can be carried out per second, so programs can run faster.

Main memory and buses

Here is a representation of the contents of main memory.

Draw a flowchart to show the process required to read the contents of memory location 0100 into the CPU.

You must include in your response:

  • the buses used
  • the contents of each bus

(6 marks)

A full mark answer would typically include points such as:

  • Load address 0100 into the MAR.
  • Address bus carries 0100 to main memory.
  • Control bus sends a read signal.
  • Data at address 0100 is placed on the data bus.
  • Data bus carries the contents back into the MDR.
  • Instruction/data is then passed to the ACC/CPU for processing.

  • Give two characteristics

  • Limited processing resources
  • Low power consumption
  • Simple/robust operating system
  • Low cost per unit
  • Smaller in size
  • Less storage
  • Limited user interface

Embedded system definition

A robotic vacuum cleaner has an embedded system.

Describe the term embedded system.

(2 marks)

Model answer (any two):

  • Computer system that is part of other equipment.
  • Performs a specific / dedicated function.
  • Includes a processor and memory.

Firmware

The robotic vacuum cleaner has firmware installed.

Define the term firmware.

(1 mark)

Answer: Software stored on ROM (read-only memory), usually providing low-level control for the device.

Processor size in embedded systems

Explain why the robotic vacuum only uses a small processor.

(4 marks)

Model points:

  • It only needs to carry out a relatively small set of instructions / simple tasks.
  • Using a small processor reduces power consumption, which is important as it runs on battery.
  • Smaller processors are cheaper, helping to keep the device affordable.
