10. Question

Discuss the differences between writing and debugging a program using an IDE and using a text editor with a separate compiler. [9]

  • IDE: integrates editor, build/compile, run and debugging tools in one environment.
  • IDE writing aids: syntax highlighting, pretty print/auto-indent, auto-complete, auto-correction.
  • IDE debugging: breakpoints, stepping/trace, watch window, error diagnostics.
  • Benefit of IDE: faster development and easier to find logical errors (step through to see variable changes).
  • Text editor + compiler: fewer built-in aids; compile and debug using separate tools.
  • Benefit of editor approach: simpler, lower memory/CPU footprint; may suit low-spec systems or quick edits.
  • Evaluation: IDE reduces errors and speeds debugging; editor can be useful for learning or when resources are limited.

function toBinary(denary)
  binaryValue = ""
  while denary > 0
    temp = denary MOD 2
    if temp == 1 then
      binaryValue = "1" + binaryValue
    else
      binaryValue = "0" + binaryValue
    endif

  return binaryValue
endfunction

 endif

Question:
Describe how low-level programming differs from procedural programming. [6]

Mark scheme points (any suitable):

  • Low-level programming is usually carried out in assembly language.
  • Assembly instructions correspond closely to machine code executed on the CPU.
  • Each assembly instruction contains an opcode and operands (e.g. addresses, values, registers).
  • Low-level programming requires careful consideration of CPU architecture / how it works.
  • Procedural programming uses high-level instructions (e.g. print), which are translated into multiple low-level instructions by a compiler.
  • High-level control structures (e.g. while / for) have clearer syntax and are generally easier to read and write.
