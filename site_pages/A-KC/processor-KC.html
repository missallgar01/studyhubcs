Question 1(a): Define the term hardware. [1]

Hardware is the physical, tangible parts of a computer system (e.g. CPU, keyboard).

Question 1(b): Define the term software. [1]

Software is the programs or instructions that control the hardware and make the computer perform tasks.

A peripheral is a device external to the main computer, connected to it, used for input, output, or storage.

Processors

Describe two special registers, other than the ALU, which are found in the processor. [4]

  • Program Counter (PC): Stores the address of the next instruction.
  • Memory Address Register (MAR): Stores the address in memory currently being accessed.
  • Memory Data Register (MDR): Stores data being transferred to or from memory.
  • Current Instruction Register (CIR): Stores the instruction currently being executed.

Explain how the Program Counter (PC) is used in the functioning of the processor. [2]

  • Stores the address of the next instruction.
  • Controls the order in which instructions are fetched and executed.
  • Is incremented after being read.
  • Is altered by jump/branch instructions.

State three different buses used in a computer system, describing what each is used for. [6]

  • Data bus: Carries data/information; bidirectional.
  • Address bus: Carries information about where data is being sent to or collected from.
  • Control bus: Carries control signals such as read/write.

Describe 4 steps in the Fetch–Decode–Execute cycle, including registers and buses. [4]

  1. Address from PC is copied to MAR.
  2. MAR sends address via the address bus.
  3. Control unit sends read signal.
  4. Data is transferred via data bus into MDR.
  5. Data is copied from MDR to CIR.

Describe what the BIOS does to start up the computer. [3]

  • Contains computer start-up instructions.
  • Loads configuration settings (CMOS/NVRAM).
  • Initialises and checks hardware/peripherals (POST).
  • Finds and loads the operating system.

Describe how an embedded system can make a different use of ROM and why it is an advantage. [2]

  • Stores firmware/configuration/operating instructions.
  • Cannot be overwritten, improving security and reliability.
  • Programs load instantly as they are already in ROM.

Question 4: Describe the role of the Control Unit (CU), Memory Unit (MU), and Arithmetic Logic Unit (ALU) within the CPU. [6]

Control Unit (CU): Manages the fetch–decode–execute cycle by sending control signals and decoding instructions.

Memory Unit (MU): Stores the operating system, application software, and data currently in use.

Arithmetic Logic Unit (ALU): Performs arithmetic operations (e.g. addition, subtraction) and logical operations (AND, OR).

Question 5: State two items that would be stored in the memory unit during processing. [2]

Examples include:

  • Part of the operating system currently in use
  • Data files currently being processed
  • Application software currently running

Question 6: Explain the role of the following registers: PC, MAR, and CIR. [3]

Program Counter (PC): Holds the address of the next instruction to be fetched.

Memory Address Register (MAR): Holds the memory address of the data or instruction to be accessed.

Current Instruction Register (CIR): Holds the current instruction while it is being decoded and executed.

Question 7: Describe the sequence of steps in the Fetch–Decode–Execute cycle. [7]

  1. The CPU copies the contents of the PC into the MAR.
  2. The address in the MAR is sent via the address bus.
  3. The CU sends a read signal via the control bus.
  4. The instruction/data is transferred to the MDR via the data bus.
  5. The instruction is copied from the MDR into the CIR.
  6. The PC increments to the next instruction.
  7. The instruction is decoded and executed.

Question 8: Explain what is meant by a bus in computer architecture. [1]

A bus is a set of parallel wires that transmits groups of bits or signals between CPU components.

Question 9: Describe the purpose of the address bus, data bus, and control bus. [6]

Address bus: Carries memory addresses of data or instructions to be accessed.

Data bus: Transfers data and instructions between the CPU, memory, and I/O devices.

Control bus: Carries control signals (e.g. read/write, clock, interrupt) from the CU.

Question 10: Distinguish between manual input methods and automatic input methods, giving one example of each. [2]

Manual input: Data entered by a user (e.g. keyboard, mouse).

Automatic input: Data entered without human involvement, usually via sensors (e.g. barcode scanner, RFID).

RAM, ROM and Virtual Memory

RAM and ROM (a): State two differences between RAM and ROM. [2]

  • RAM is volatile and loses its contents when power is off, whereas ROM is non-volatile.
  • RAM can be read from and written to, whereas ROM can normally only be read from.

RAM and ROM (b): Explain why RAM is required in a general-purpose computer system. [2]

  • Stores programs, data, and the operating system currently in use.
  • Provides fast access for the processor to working data.

RAM and ROM (c): State one example of software stored in ROM. [1]

Bootstrap program / BIOS.

Embedded Systems (a): Explain why the control software is stored in ROM. [3]

  • The control software does not need to be changed.
  • ROM cannot normally be altered by the user.
  • The program is immediately available when powered on.

Embedded Systems (b): Explain why it is still necessary for the device to have some RAM. [3]

  • Provides working memory for the processor.
  • Acts as a buffer when loading or playing back TV signals.
  • Stores data currently being processed or running.

ROM and Start-Up: Describe three tasks carried out by software stored in ROM during start-up. [3]

  • Loads configuration settings (e.g. CMOS/NVRAM).
  • Initialises and checks hardware and peripherals (POST).
  • Reports hardware errors.
  • Determines which drive contains the OS.
  • Loads the bootstrap / OS kernel into RAM.

Virtual Memory (a): Explain what is meant by virtual memory. [2]

  • Uses secondary storage (e.g. hard disk/SSD) to extend RAM.
  • Stores data and instructions that do not fit into physical RAM.

Virtual Memory (b): Describe how virtual memory allows a large program to run. [2]

  1. Inactive sections of programs/data are swapped out to secondary storage.
  2. Active sections are swapped into RAM when required.

Virtual Memory (c): State one disadvantage of using virtual memory. [2]

  • Slower access speed compared to RAM.
  • May cause disk thrashing if excessive swapping occurs.

Features of Von Neumann and Contemporary Processors

Von Neumann Architecture (a): State two features of the Von Neumann architecture. [2]

  • Instructions and data are stored in the same memory.
  • Instructions and data share the same buses.
  • Single control unit and single ALU.
  • Uses the fetch–decode–execute cycle.

Von Neumann Architecture (b): Explain one limitation of the Von Neumann architecture. [2]

Only one fetch can occur at a time, so the CPU cannot access instructions and data simultaneously.

This causes the Von Neumann bottleneck, limiting performance.

Von Neumann Architecture (c): Give one example of where a Von Neumann architecture might be used. [1]

General-purpose computer such as a PC or laptop.

Harvard Architecture (a): Describe the Harvard architecture. [2]

  • Physically separate memories for instructions and data.
  • Separate buses allow concurrent access.

Harvard Architecture (b): State one advantage of the Harvard architecture over Von Neumann. [1]

Instructions and data can be accessed at the same time, improving speed.

Harvard Architecture (c): Explain why the Harvard architecture is often used in embedded systems. [3]

  • Instructions can be stored in ROM while data is stored in RAM.
  • Provides fast and predictable performance.
  • Avoids the Von Neumann bottleneck and is efficient for dedicated devices.

Discuss the use of different computer architectures for different problem solutions. [8]

Von Neumann Architecture

  • Data and instructions stored together in memory.
  • Uses sequential processing following an algorithm.
  • Suitable where steps must occur in a strict order.
  • Suffers from performance limitations due to single processor.

Parallel Processing

  • Uses multiple processors or cores.
  • Executes instructions at the same time.
  • Requires complex coordination.
  • Often uses co-processors such as GPUs.

Contemporary Processors (a): State two features that improve performance. [2]

  • Cache memory
  • Pipelining
  • Multiple cores
  • Hyper-threading / virtual cores

Contemporary Processors (b): Explain the role of cache memory in a CPU. [2]

Cache is small, high-speed memory close to the CPU.

It stores frequently used data and instructions, reducing access to slower RAM.

Contemporary Processors (c): Explain how pipelining improves CPU performance. [2]

  • Allows overlapping of fetch, decode, and execute stages.
  • Increases instruction throughput.

Contemporary Processors (d): State one difference between a single-core and a multi-core processor. [2]

A single-core processor executes one instruction stream at a time, whereas a multi-core processor can execute multiple instruction streams concurrently.

RISC vs CISC (a): State two features of RISC instruction sets. [2]

  • Each instruction executes in a single clock cycle.
  • Uses simple instructions of fixed length.
  • Requires more RAM as programs use more instructions.

RISC vs CISC (b): State two features of CISC instruction sets. [2]

  • Single instructions may take multiple clock cycles.
  • Uses complex instructions that may perform multiple operations.
  • Requires less RAM as fewer instructions are needed.

RISC vs CISC (c): Explain why RISC processors are often preferred in mobile and embedded devices. [2]

  • More power-efficient due to simpler circuitry.
  • Fast execution of simple instructions suits real-time processing.

RISC vs CISC (d): Explain why CISC processors are often used in general-purpose desktop computers. [2]

  • Complex instructions make programs simpler and easier to write.
  • Compiler has less work translating high-level code.

Comparison (a): Compare the use of a Von Neumann architecture processor with a Harvard architecture processor. [4]

  • Von Neumann uses the same memory and buses for instructions and data.
  • Harvard uses separate memory and buses for instructions and data.
  • Von Neumann suffers from a bottleneck as only one fetch can occur at a time.
  • Harvard can fetch instructions and data simultaneously, so is faster.
  • Von Neumann is common in PCs; Harvard is common in embedded/DSP systems.

Comparison (b): Compare the use of a RISC processor with a CISC processor, giving one application suited to each. [4]

  • RISC uses simple instructions, one per cycle, is power-efficient, but needs more RAM.
  • CISC uses complex instructions, fewer per program, but may take multiple cycles.
  • RISC is suited to embedded/mobile systems.
  • CISC is suited to general-purpose desktop computers.

Storage

Long Answer Question: Modern computers tend to have magnetic or solid-state (flash) hard drives. Discuss which hard drive you would recommend for a keen video games player to use on their desktop PC. [9]

Knowledge and Understanding

  • Magnetic hard drives work by magnetic patterns being read off platters that mechanically spin at high speeds.
  • Flash (solid-state) drives use memory chips and have no moving parts.
  • Magnetic hard drives are cheaper per GB and are available in much larger capacities.
  • Flash drives have much higher read/write speeds than magnetic hard disks.
  • Flash drives consume less power and operate silently.

Application

  • Modern games often include large amounts of media, so a gamer needs significant storage space.
  • Games are fast-paced and competitive, so high loading speeds are beneficial.
  • Gamers typically prioritise high-performance components.
  • Magnetic drives can be noisy, whereas solid-state drives operate silently, improving the gaming experience.
  • Hybrid approaches exist that combine large capacity with faster access speeds.

Evaluation

Magnetic hard drives offer excellent value due to their large storage capacities at a low cost, allowing gamers to install many games at once. However, their slower performance can increase loading times.

Solid-state drives provide much faster read/write speeds, resulting in quicker game loading and smoother performance, which is highly desirable for gaming. Although SSDs are more expensive per GB, their speed advantage makes them the better choice for a keen gamer.

Overall, a solid-state drive is the recommended choice for a gaming desktop, potentially combined with a magnetic drive for additional storage.
