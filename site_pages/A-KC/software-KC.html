Topic A – System & Application Software

1. System software vs application software

(a) What is meant by system software?
(b) What is meant by application software?

(a) System software is a set of programs that manage the hardware of the system and enable the machine to perform tasks requested by the user.

(b) Application software is programs that perform specific user-oriented tasks, such as running a payroll, writing a letter or editing photos.

7. What is utility software & lack of RAM

(a) Describe what is meant by utility software.
(b) Describe a technique operating systems can use to overcome a lack of available RAM.

(a) Utility software is a program (or set of programs) with a single, specific purpose, usually related to the upkeep or maintenance of a computer system (e.g. antivirus, backup, disk cleanup).

(b) Use of virtual memory with paging:

  • Memory contents are divided into fixed-size pages.
  • Pages that are not currently needed are moved to an area of secondary storage used as virtual memory.
  • When required, those pages are swapped back into RAM so large programs can run despite limited physical memory.

Topic C – Operating Systems

9. Describing an operating system

Describe what is meant by an operating system.

An operating system is a set of programs designed to manage the hardware of a system and provide a platform on which application software can run.

10. Purposes / functions of an operating system

Give three different functions of an operating system.

Any three, e.g.:

  • Provide a user interface.
  • Manage hardware / devices (peripherals).
  • Manage memory.
  • Manage CPU usage and scheduling.
  • Provide system security and user accounts.
  • Provide utilities / housekeeping software.
  • Provide a platform for installing and running other software.
  • Handle interrupts.

11. Single-user operating system

Describe a single-user operating system.

  • Allows only one user at a time to use the system.
  • Allocates each user rights and keeps their files separate.

12. Multitasking operating system

Describe a multitasking operating system.

  • Allows apparently more than one task / program to run at the same time.
  • Gives each task a share of processor time before moving to the next.
  • Often uses separate windows for each task (e.g. listening to music while typing an essay).

13. Multi-user operating system

Describe a multi-user operating system and give an example of where it might be used.

  • One powerful computer with many terminals; more than one user can use the system at the same time.
  • Each terminal is given a time slice in turn; time slices are very small (fractions of a second).
  • System uses priorities / privileges and security to keep users’ data separate.
  • Example: supermarket checkout system, online gaming server, or mainframe with many terminals.

14. Real-time operating systems

(a) State what is meant by a real-time operating system.
(b) Explain why a real-time OS is suitable for Intensive Care Units where patients’ vital signs are monitored.

(a) A real-time OS guarantees that it will respond to inputs within a fixed maximum time.

(b) If something happens to a patient, the system must respond immediately (e.g. sounding an alarm). Other types of system may have delays, which could mean treatment is not given in time and the patient is put at risk.

Any two, e.g.:

  • Control access so different users can have their own accounts, saves and achievements.

16. Distributed operating systems & embedded systems

(a) Explain why a CGI company with many high-performance computers might use a distributed operating system.
(b) A digital photo frame displays photos automatically. What type of operating system is it likely to use?

Topic D – Memory Management & Virtual Memory

17. Paging and segmentation

(a) Explain what is meant by paging.
(b) Explain what is meant by segmentation.
(c) State two similarities between paging and segmentation.

(a) Paging divides memory into equal-size physical blocks called pages. Parts of a program are split into pages that fit these fixed-size blocks.

(b) Segmentation divides memory into unequal logical divisions called segments, such as code, data and stack, whose sizes depend on the program structure.

(c) Similarities:

  • Both are ways of partitioning / splitting up memory.
  • Both can be used with virtual memory, storing parts of programs on backing store and loading them into RAM when needed.
  • Both allow programs larger than main memory to run and can store programs non-contiguously in memory.

18. Virtual memory & disk thrashing

(a) Why is virtual memory needed?
(b) Describe how virtual memory is used.
(c) Describe the problem of disk thrashing.

(a) Virtual memory is needed to allow programs to run that require more memory than is physically available.

(b) An area of backing store (e.g. hard disk) is used as if it were main memory. The OS uses paging to swap pages between RAM and this storage so that pages currently needed are in RAM and others are held on disk.

(c) Disk thrashing occurs when so many page swaps are taking place that the computer spends more time transferring pages between RAM and disk than doing real processing. Because the disk is relatively slow, performance collapses and the system may appear to hang.

19. Smartwatch OS – virtual memory scenario

A smart-watch OS uses memory management.

(a) List three functions of an operating system other than memory management.
(b) The OS divides memory into equally sized chunks as shown in a diagram. What name is given to this type of memory management?
(c) The OS needs to load program C but there is not enough space. Describe how virtual memory would be used to load program C.

(a) Any three of: provide a user interface; manage hardware/peripherals; manage CPU usage / handle interrupts; provide security; provide a platform for other software; provide utilities.

(b) Paging.

(c) The OS uses part of secondary storage as virtual memory. It moves unused pages or parts of existing programs (e.g. A and/or B) from RAM to this area, freeing space in physical memory. It then loads pages of program C into the freed frames.

20. Why memory management is necessary

Explain why memory management is necessary in an operating system.

  • Organises the use of main memory by converting logical addresses to physical addresses.
  • Allows multiple programs to share memory safely and protects them from accessing each other’s data.
  • Enables programs larger than physical memory to run by using virtual memory.
  • Ensures efficient use of limited RAM so the system remains responsive and secure.

21. Extended question – Memory management (9 marks)

Exam-style extended question:
Memory management is a key function of an operating system. Explain how an operating system can manage the memory available to applications and why doing so is important.

Key points for a high-level answer:

  • OS allocates and deallocates RAM to processes; keeps track of which areas are free / in use.
  • Uses paging (fixed-size physical divisions) and/or segmentation (variable-size logical divisions) to split memory.
  • Maintains page/segment tables to map logical addresses to physical locations.
  • Uses virtual memory when RAM is nearly full – moves pages to backing store and swaps them back when needed.
  • Prevents one process accessing memory reserved for another (security and stability).
  • Frees memory when programs finish or no longer need it (garbage collection / deallocation).
  • Discuss advantages and issues: allows multitasking, efficient use of RAM, but excessive use of virtual memory leads to slow-down and disk thrashing.
  • Conclude that good memory management is essential for an efficient, stable and secure system.

Topic E – CPU Scheduling

22. Why scheduling is used

Explain why operating systems use scheduling.

  • To ensure all tasks are processed fairly by sharing CPU time.
  • To process as many jobs as possible in the least possible time (maximise throughput).
  • To maximise the number of interactive users receiving fast response times.
  • To obtain efficient use of processor time and other resources, and prevent any job from monopolising the CPU.

23. Round Robin scheduling

(a) Describe how the Round Robin scheduling algorithm works.
(b) Give one advantage and one disadvantage of Round Robin.

(a) Each ready job/process is given a fixed time slice of CPU time in turn. When its time slice expires, it is moved to the back of the queue and the next job gets the CPU. This repeats until all jobs are finished. Priorities may also be used.

(b) Advantage: Fair – all processes get a regular share of CPU; good for time-sharing systems; users see little apparent delay.
Disadvantage: If the time slice is too small there will be lots of context switching and wasted CPU time; long jobs may take a long time to complete.

24. First Come, First Served and other algorithms

(a) Explain why the First Come, First Served scheduling method may not be efficient.
(b) Describe one other scheduling method.
(c) Summarise why scheduling is necessary in general.

(a) Once a job starts it prevents other jobs from being processed until it finishes. If a long job or one using a slow resource (e.g. printer) is first, it can waste processor time and make other jobs wait a long time.

(b) Example – Shortest Job First: the job with the shortest total processing time is run next; or Round Robin; or Multi-Level Feedback Queues; or priority scheduling. (Description required.)

(c) Scheduling is needed to:

  • Process as many jobs as possible in the least possible time.
  • Ensure all jobs are processed and resources are used efficiently.
  • Give many interactive users quick response times.

25. Extended question – Managing CPU time & resources (9 marks)

Exam-style extended question:
One key function of an OS is to manage system resources such as memory and CPU time. Discuss different approaches a desktop OS can take to managing these resources and the advantages of each approach.

Key points:

  • Explain that CPU time is shared via scheduling algorithms (Round Robin, FCFS, Shortest Job First, Shortest Time Remaining, Multi-Level Feedback Queues).
  • Discuss how each algorithm works and its pros/cons (e.g. fairness vs efficiency; risk of starvation; context-switch overhead).
  • Link to memory management approaches (paging, segmentation, virtual memory) and how they allow many programs to run together.
  • Explain importance of interrupts so the CPU can respond quickly to I/O or urgent events instead of polling.
  • Evaluate: more complex algorithms give better responsiveness but use more CPU time; real-time systems require guaranteed response; trade-offs between throughput, fairness and complexity.

Topic F – Interrupts & the CPU

26. What is an interrupt?

(a) State what is meant by the term interrupt.
(b) Explain why unplugging an external hard drive may need to interrupt the current fetch-decode-execute cycle.

(a) An interrupt is a signal to the processor indicating that a device or process needs attention.

(b) Removing a drive means the OS must immediately stop transferring data, close open files and update the file system. The CPU has to be interrupted so it can safely stop using the device before it is removed.

27. How the CPU handles an interrupt

Describe what happens inside the CPU when it receives an interrupt.

  • The CPU checks for interrupts at the end (or start) of each fetch-execute cycle.
  • If the interrupt is lower or equal priority to the current process, the current process continues.
  • If it is higher priority, the CPU finishes the current instruction, then copies the contents of its registers to a stack in memory.
  • Flags are set as appropriate; the program counter is loaded with the address of the relevant Interrupt Service Routine (ISR).
  • The ISR runs to deal with the event.
  • When the ISR finishes, the previous register values are popped from the stack and restored so the interrupted program can resume where it left off.

28. Using a stack when handling interrupts

Explain how a stack data structure is used when handling interrupts.

  • A stack is a LIFO / FILO structure.
  • When an interrupt occurs, the current register contents (and possibly return address) are pushed onto the stack.
  • When the ISR completes, these values are popped from the stack in reverse order so the CPU can return to the interrupted process correctly.

Topic H – BIOS & Boot Process

30. BIOS and booting the OS

(a) Describe what is meant by the term BIOS.
(b) Describe how a desktop operating system is loaded when a PC is first switched on.

(a) The BIOS (Basic Input/Output System) is firmware stored in ROM that tests hardware components (POST) and boots/loads the operating system.

(b) When the computer is powered on, the BIOS is executed from ROM, performs its checks (POST), then loads the bootstrap loader which in turn loads the operating system from secondary storage into main memory.

31. Effects of overwriting the BIOS

A virus overwrites and destroys the contents of a computer’s BIOS.

Describe the effect this would have on the computer.

  • The computer would no longer be able to boot / load the operating system.
  • It would be unable to carry out initial hardware checks or configuration.
  • In practice the machine would be unusable until the BIOS is repaired or replaced.

Topic I – Virtual Machines

32. Virtual machines – definition & uses

(a) Describe what is meant by a virtual machine.
(b) A programmer is writing a game to run on several different consoles. Discuss ways they might use virtual machines in this process and the advantages and disadvantages of doing so.

(a) A virtual machine is software that behaves like a complete computer system, emulating hardware so it can run its own operating system and applications.

(b) Key discussion points:

  • VMs of different consoles allow the programmer to test the game for each platform on one development machine instead of needing multiple physical consoles.
  • VMs can provide safe, isolated environments for testing possibly unstable builds.
  • The game could be written in a language that compiles to intermediate code which runs on a VM; the same code can then run on different hardware with suitable VMs.
  • This saves time and cost (one codebase for many platforms) but the game may run more slowly than native compiled code.
  • VMs require a powerful host machine and must accurately emulate the target hardware; otherwise the final code may not behave the same on real consoles.
